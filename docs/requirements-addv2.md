## SEO改善タスクリスト（優先度順）

### P0: メタ情報とドメイン整合性の是正
- `og:title` / `og:description` / `og:image` および `twitter:card` 系メタタグを追加し、検索結果やSNS共有で視認性が落ちないようにする。特に代表画像は `https://f1weathers.com/og-default.jpg`（仮）など公開済みのパスを用意し、ビルド時に参照漏れがないか確認する。
- `<link rel="canonical">` を `https://f1weathers.com/` に設定し、旧ドメイン（`f1-weather.vercel.app`）へのアクセスは恒久的な 301 リダイレクトで統一する。手動確認としてブラウザの開発者ツールでステータスコードを検証し、Search Console の「インデックス カバレッジ」で重複URLが解消されたことを追跡する。
- `application/ld+json` の `url` を `https://f1weathers.com` に修正し、同時に `@id` を付与してサイト全体のリファレンスを固定する。構造化データテストツールでエラーがないことを確認する。

### P1: 構造化データの拡張とサイトマップ最適化
- レースごとに `SportsEvent` スキーマを生成し、開催日時（`startDate` / `endDate`）、開催地（`location`）、気温や降水確率のハイライト（`eventStatus` や `additionalProperty`）を含める。週次バッチで最新予報を反映し、更新日時を `dateModified` に記録する。
- `public/sitemap.xml` を動的生成に切り替え、全レース詳細ページや主要ドキュメントを網羅する。更新頻度・優先度を実際の更新サイクルに合わせ、CIでビルド時に再生成する仕組みを整える。

### P2: 情報設計とコンテンツ拡充
- H1 を単一に整理し、H2 以降をレースカテゴリ（例：今後のレース、アーカイブ）で階層化する。準備中セクションでは H2 → H3 の順でサーキット名を示し、「天気予報は準備中です」といった補足は段落や説明リストで表現する。スクリーンリーダーで適切に読み上げられることを確認する。
- 開催地ごとの気候特性、持参すべき装備、過去の天候傾向など、検索意図に応じた固有テキストを追記する。更新フローでは気象データの出典と検証手順を `docs/` 配下に整理し、編集ガイドラインを共有する。
- トップページから各レース詳細ページ、`docs/` 内の関連資料、外部の公式情報への内部リンクを整備し、パンくずリストの導入を検討する。Google Search Console の「リンク」レポートで内部リンクが増加していることを観測する。

### P3: 多言語展開と地域ターゲティング
- 英語版など多言語化を行う場合は、Next.js の国際化ルーティングを活用し `hreflang` を適切に出力する。対象市場（例：英語、スペイン語）ごとにキーワードリストを作成し、翻訳ではなくローカライズを意識する。
- 地域別のモバイルユーザー体験を向上させるため、`Core Web Vitals` の定期計測を `PageSpeed Insights` / `Lighthouse CI` で実施し、CLS や LCP の改善を継続する。AMP 化は現状不要だが、F1開催週の急増するトラフィックに備えたキャッシュ戦略（CDNのエッジキャッシュ更新タイミング）を検討する。
